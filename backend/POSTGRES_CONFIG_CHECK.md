# Проверка конфигурации PostgreSQL

## ✅ Видимые настройки (правильно):

1. **Локаль сортировки:** `C` ✅
2. **Локаль набора символов:** `C` ✅
3. **Сеть:** `default` ✅
4. **Группы безопасности:** `default-sg-...` ✅
5. **Хост:** Добавлен в `ru-central1-a` с публичным доступом ✅
6. **Диагностика:** Стандартный режим ✅
7. **Резервное копирование:** 22:00-23:00 UTC ✅
8. **Срок хранения:** 7 дней ✅

## ⚠️ Что нужно проверить/заполнить:

### Основные параметры (в начале формы):

1. **Имя кластера:** `happyrussia-db`
2. **Окружение:** `Production` или `Testing`
3. **Версия PostgreSQL:** `15` (или последняя доступная)
4. **Класс хоста:** 
   - Для тестов: `s2.micro` (1 vCPU, 4 GB) - дешевле
   - Для продакшена: `s2.small` (2 vCPU, 8 GB) - надежнее
5. **Размер диска:** `10 GB` (минимум) или больше
6. **Тип диска:** `SSD`

### База данных:

1. **Имя БД:** `happyrussia`
2. **Имя пользователя:** `happyrussia_user`
3. **Пароль:** Сгенерируйте надежный пароль (сохраните!)

### Важно:

- **Стоимость:** 6 190 ₽/мес - это для класса хоста выше минимального
- Если хотите сэкономить, выберите `s2.micro` - будет дешевле (~1500-2000 ₽/мес)
- Публичный доступ включен ✅ - это правильно для доступа из контейнера

## После создания:

1. Дождитесь статуса "Running" (5-10 минут)
2. Откройте кластер → "Подключение"
3. Скопируйте FQDN хоста
4. Обновите контейнер с переменной `DATABASE_URL`
