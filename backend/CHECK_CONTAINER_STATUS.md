# Проверка статуса контейнера

## Проблема
Запросы к API выполняются очень долго или таймаутят.

## Возможные причины

1. **Cold Start** - Serverless Containers могут иметь задержку при первом запросе (до 30-60 секунд)
2. **Контейнер еще не полностью запустился**
3. **Проблемы с базой данных** - SQLite может быть недоступен в контейнере

## Решения

### 1. Проверить логи контейнера

**Через веб-консоль:**
1. Откройте контейнер `happyrussia-api`
2. Перейдите на вкладку **"Логи"**
3. Проверьте, есть ли ошибки

**Через CLI:**
```bash
yc serverless container logs --name happyrussia-api
```

### 2. Проверить статус ревизии

**Через веб-консоль:**
1. Откройте контейнер
2. Перейдите на вкладку **"Ревизии"**
3. Проверьте статус последней ревизии (должен быть "Активна")

### 3. Проблема с базой данных

SQLite может не работать в Serverless Containers из-за:
- Отсутствия постоянного хранилища
- Проблем с правами доступа

**Решение:** Использовать Managed PostgreSQL или настроить volume для данных.

### 4. Увеличить таймаут

Если контейнер долго стартует, можно:
- Увеличить таймаут выполнения до 60 секунд
- Увеличить память до 1GB

### 5. Проверить через браузер

Попробуйте открыть в браузере:
```
https://bbas207e8gbhlpbjb51u.containers.yandexcloud.net/api/health
```

## Временное решение

Если контейнер не работает, можно временно использовать локальный бекенд для разработки, а для продакшена настроить PostgreSQL.
