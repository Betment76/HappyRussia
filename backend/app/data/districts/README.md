# Данные по федеральным округам

Данные о населенных пунктах России разбиты на 8 файлов по федеральным округам для оптимизации производительности.

## Структура файлов

Файлы используют безопасные имена (латиница) для совместимости с разными системами:

- `central.json` (Центральный) - 18 регионов, 2.17 MB
- `northwest.json` (Северо-Западный) - 11 регионов, 2.54 MB
- `south.json` (Южный) - 13 регионов, 996 KB
- `north_caucasus.json` (Северо-Кавказский) - 7 регионов, 624 KB
- `volga.json` (Приволжский) - 14 регионов, 3.41 MB
- `ural.json` (Уральский) - 6 регионов, 40 KB
- `siberian.json` (Сибирский) - 10 регионов, 1.02 MB
- `far_east.json` (Дальневосточный) - 11 регионов, 717 KB

## Формат файла

Каждый файл содержит данные одного федерального округа:

```json
{
  "name": "Название округа",
  "population": 0,
  "regions": [
    {
      "id": "код региона",
      "name": "Название региона",
      "population": 0,
      "federal_district": "Название округа",
      "cities": [...],
      "urban_districts": [...]
    }
  ]
}
```

## Загрузка данных

Данные автоматически загружаются функцией `load_russia_data()` из `russia_settlements.py`.

Если файлы в этой папке не найдены, система попытается загрузить данные из старого файла `settlements_data.json`.

## Преимущества

1. **Производительность IDE** - небольшие файлы не тормозят редактор
2. **Быстрая загрузка** - загружаются только нужные округа
3. **Удобство работы** - можно редактировать данные по округам отдельно
4. **Совместимость** - имена файлов на латинице работают на всех системах (Windows, Linux, macOS)
5. **Безопасность** - нет проблем с кодировкой при передаче файлов или работе с Git

## Обновление данных

Для обновления данных используйте скрипты из `backend/scripts/`:
- `split_settlements_by_districts.py` - разбить большой файл на округа (создает файлы с безопасными именами)
- `rename_districts_to_safe_names.py` - переименовать файлы в безопасные имена (если нужно)
- `update_*.py` - обновить данные конкретного региона

## Почему латиница?

Имена файлов используют латиницу вместо кириллицы по следующим причинам:
- ✅ Работает на всех операционных системах без проблем с кодировкой
- ✅ Совместимо с Git и системами контроля версий
- ✅ Нет проблем при передаче файлов между системами
- ✅ Удобно для веб-серверов и контейнеров
- ✅ Код автоматически преобразует русские названия в имена файлов
