# Настройка публичного доступа к контейнеру

## Проблема
Контейнер возвращает 403 Forbidden при обращении без аутентификации.

## Решение: Настроить публичный доступ

### Через веб-консоль:

1. Перейдите в [Serverless Containers](https://console.cloud.yandex.ru/folders)
2. Откройте контейнер `happyrussia-api`
3. Перейдите на вкладку **"Ревизии"**
4. Откройте активную ревизию
5. Найдите раздел **"Доступ"** или **"Публичный доступ"**
6. Включите **"Публичный доступ"** или **"Unauthenticated access"**
7. Сохраните изменения

### Альтернатива: Настроить через IAM

Если публичный доступ недоступен, можно:

1. Создать новый API Gateway перед контейнером
2. Или использовать Application Load Balancer
3. Или настроить аутентификацию в приложении

## Проверка после настройки

После включения публичного доступа проверьте:

```bash
curl https://bbas207e8gbhlpbjb51u.containers.yandexcloud.net/api/health
```

Должен вернуть: `{"status":"ok"}`

## Если публичный доступ недоступен

Можно временно использовать локальный бекенд для разработки, а для продакшена настроить API Gateway или Load Balancer.
