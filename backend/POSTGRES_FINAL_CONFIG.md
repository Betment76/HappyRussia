# ✅ Финальная конфигурация PostgreSQL

## Текущие настройки (правильно):

- ✅ **Окружение:** `PRODUCTION`
- ✅ **Версия:** `15`
- ✅ **Тип:** `standard` (100% vCPU) - правильно для продакшена
- ✅ **Класс хоста:** `s1.micro` (2 cores, 100% vCPU, 8 GB RAM)
- ✅ **Тип диска:** `network-hdd`
- ✅ **Размер диска:** `10 GB`
- ✅ **Стоимость:** 6 302,30 ₽/мес

## Что еще нужно заполнить:

### 1. Основные параметры (в начале формы):
- **Имя кластера:** `happyrussia-db`
- **Описание:** (опционально) "База данных для приложения HappyRussia"

### 2. База данных (секция "База данных"):
- **Имя БД:** `happyrussia`
- **Имя пользователя:** `happyrussia_user`
- **Пароль:** Сгенерируйте надежный пароль (сохраните!)

### 3. Сеть (если еще не настроено):
- **Сеть:** `default` (или выберите нужную)
- **Публичный доступ:** ✅ Включить (для доступа из контейнера)

### 4. Безопасность:
- Добавьте правило безопасности для доступа из контейнера:
  - **Источник:** `0.0.0.0/0` (или конкретный IP)
  - **Тип:** `CIDR`

## Рекомендации по оптимизации стоимости:

Если хотите сэкономить, можно:
1. **Изменить окружение на TESTING:**
   - Стоимость снизится примерно на 20-30%
   - Для начала этого достаточно

2. **Использовать более дешевый класс:**
   - `s2.micro` (1 vCPU, 4 GB) - дешевле
   - Но текущий `s1.micro` (2 vCPU, 8 GB) - лучше для продакшена

3. **Тип диска:**
   - `network-hdd` - дешевле (текущий выбор)
   - `network-ssd` - быстрее, но дороже

## После создания:

1. Дождитесь статуса **"Running"** (5-10 минут)
2. Откройте кластер → вкладка **"Подключение"**
3. Скопируйте **FQDN хоста** (например: `c-xxxxx.rw.mdb.yandexcloud.net`)
4. Запомните порт: `6432`
5. Обновите контейнер с переменной `DATABASE_URL`

## Строка подключения (после создания):

```
postgresql://happyrussia_user:ВАШ_ПАРОЛЬ@c-xxxxx.rw.mdb.yandexcloud.net:6432/happyrussia
```
