# ⚠️ Предупреждение о конфигурации PostgreSQL

## Проблема

Вы выбрали:
- **Окружение:** `PRODUCTION` 
- **Тип хоста:** `burstable` (50% vCPU)
- **Класс:** `b1.medium` (2 cores, 50% vCPU, 4 GB)

**Предупреждение:** Хосты с гарантированной долей vCPU ниже 100% рекомендуются только для сред тестирования.

## Решение

### Вариант 1: Изменить окружение на Testing (рекомендуется для начала)

1. Измените **Окружение** на `TESTING`
2. Оставьте `burstable` тип
3. Это подходит для тестирования и разработки
4. Стоимость: ~2 787 ₽/мес

### Вариант 2: Изменить тип хоста на standard (для продакшена)

1. Оставьте **Окружение** `PRODUCTION`
2. Измените **Тип** на `standard` (100% vCPU)
3. Выберите класс: `s2.micro` (1 vCPU, 4 GB) или `s2.small` (2 vCPU, 8 GB)
4. Это даст стабильную производительность
5. Стоимость: ~1 500-3 000 ₽/мес (зависит от класса)

### Вариант 3: Оставить как есть (не рекомендуется)

Если оставить `PRODUCTION` + `burstable`:
- ⚠️ Может быть нестабильная производительность
- ⚠️ Подходит только для тестирования
- ✅ Дешевле

## Рекомендация

Для начала используйте:
- **Окружение:** `TESTING`
- **Тип:** `burstable` 
- **Класс:** `b1.medium` (текущий)
- **Диск:** `network-hdd` 10 GB (можно оставить или изменить на SSD)

Это оптимально для тестирования и разработки. Позже можно перейти на `PRODUCTION` с `standard` типом.

## Дополнительные настройки

### Тип диска:
- **network-hdd** - дешевле, медленнее
- **network-ssd** - дороже, быстрее (рекомендуется для продакшена)

### Размер диска:
- **10 GB** - минимум, подходит для начала
- Можно увеличить позже

## Что дальше
