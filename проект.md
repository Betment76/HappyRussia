Общая информация
Приложение "Счастливый день" разрабатывается на Flutter для кроссплатформенной поддержки Android и iOS. Оно позволяет пользователям ежедневно фиксировать настроение выбором одного из 5 смайликов (от грустного до веселого) с привязкой к региону России. Главный экран отображает все регионы РФ, отсортированные по среднему уровню "счастливости" на основе агрегированных данных пользователей.​

Цели и задачи
Цели:

Собрать анонимные данные о настроении пользователей по регионам России для визуализации коллективного эмоционального состояния.

Мотивировать ежедневные чекины через простой интерфейс, аналогичный трекерам настроения вроде Moodistory.​

Предоставить пользователям обзор "счастливости" регионов для повышения осведомленности о социальном настроении.

Задачи:

Реализовать ежедневный выбор настроения с геолокацией или ручным выбором региона.

Отображать динамический рейтинг регионов на главном экране.

Обеспечить backend для хранения и агрегации данных.​

Функциональные требования
Ежедневный чек-ин: Пользователь выбирает 1 из 5 смайликов (1 — грустный, 5 — веселый). Автоматическое определение региона по GPS или выбор из списка 85 субъектов РФ. Данные отправляются на сервер анонимно ежедневно.

Главный экран: Список регионов, отсортированный по убыванию среднего балла "счастливости" (за день/неделю/месяц). Каждый элемент — название региона, средний смайлик, прогресс-бар или цветовой индикатор.​

Дополнительно: Статистика своего региона/страны, история личных чек-инов, напоминания о ежедневном выборе, переключение периодов просмотра.

Авторизация: Опциональная (для истории), анонимная для чек-инов.

Экран	Основные элементы	Действия
Главный	Список регионов с рейтингом	Фильтр по периоду, обновление
Чек-ин	5 смайликов, выбор региона	Подтверждение, отправка данных
Профиль	Личная история, статистика	Просмотр графика настроения ​
Нефункциональные требования
UI/UX: Минималистичный дизайн с яркими смайликами, адаптивный под светлую/темную тему. Использовать Flutter widgets: ListView для регионов, GestureDetector для смайликов.​

Производительность: Оффлайн-доступ к последним данным, синхронизация при подключении. Загрузка списка регионов <1 сек.

Безопасность: Анонимность данных, GDPR-соответствие, хранение только агрегированных статистик.

Backend: YandexCloud ​

Тестирование: Unit-тесты для логики сортировки, интеграционные для API.

Техническая реализация
Стек: Flutter 3.x (Dart), Provider/Riverpod для состояния, Geolocator для локации, HTTP/Dio для API.

Структура проекта: lib/screens (главный, чек-ин), lib/models (RegionMood, MoodLevel), lib/services (api, storage).
